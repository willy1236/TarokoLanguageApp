#### A1. `words`（單字，1,087 row）
```sql
CREATE TABLE words (
  id            SERIAL PRIMARY KEY,
  truku         TEXT NOT NULL,                          -- 太魯閣羅馬拼音 "huling"
  zh            TEXT NOT NULL,                          -- "狗"
  en            TEXT,                                   -- 部分有

  level         TEXT NOT NULL,                          -- 4 級 enum

  explanation   TEXT,                                   -- 預留，目前空
  source_module TEXT NOT NULL DEFAULT 'vocabulary',    -- ETL metadata
  source_no     TEXT NOT NULL,                          -- klokah 編號 "07-01"
  source_subcat TEXT,                                   -- 原 36 細分類

-- enum: category ∈ {nature, food, people, life, culture, action, abstract, grammar}
```

#### A4. `audio_index`（音檔路徑索引，3,484 row）
- 保留作 ETL 對帳；**App 流程未使用**（已改 GCS 直連）

#### B. `users`（認證，0 row）
```sql
CREATE TABLE users (
  uid                  SERIAL PRIMARY KEY,
  provider             TEXT NOT NULL,                   -- 'google' | 'apple'
  provider_sub         TEXT NOT NULL,                   -- Firebase UID, API 永不回傳
  display_name         TEXT,
  avatar_url           TEXT,
  email                TEXT,                            -- 僅本人 GET /api/me 可見
  token_invalidated_at TIMESTAMPTZ,                     -- JWT 撤銷時間
  created_at           TIMESTAMPTZ DEFAULT NOW(),
  updated_at           TIMESTAMPTZ DEFAULT NOW(),
  last_login_at        TIMESTAMPTZ,
  UNIQUE (provider, provider_sub)
);
```

#### C1. `quiz_sessions` / `quiz_questions`（單字測驗）
```sql
quiz_sessions:    id UUID PK, user_uid, level, status, score 0-10, started_at, last_active_at, completed_at
quiz_questions:   id UUID PK, session_id FK CASCADE, question_order 1-10,
                  correct_word_id → words.id, option_ids INT[4],
                  direction ∈ {zh_to_truku, truku_to_zh},
                  user_answer_id, is_correct
                  CHECK array_length(option_ids,1)=4
                  CHECK correct_word_id = ANY(option_ids)
                  UNIQUE (session_id, question_order)
```

#### C2. `listening_sessions` / `listening_questions`（聽力測驗）
```sql
listening_sessions:    比 quiz_sessions 多一欄 mode ∈ {word_to_zh, word_to_truku, sentence_to_zh}
listening_questions:   correct_word_id 或 correct_sentence_id 二擇一（依 mode）
                       其餘同 quiz_questions
```

### 3.4 約束概覽
- 所有 `level` 都用同一份 enum：`('初級','中級','中高級','高級')`
- 所有 `status` 都用：`('in_progress','completed')`
- `option_ids` 強制 4 元素、且 `correct_*_id` 必須在內
- `(session_id, question_order)` 唯一
- `users (provider, provider_sub)` 唯一

## 4. API endpoints × DB 操作對照

> 規格詳見 [`API設計/資料交換表_核心.md`](../API設計/資料交換表_核心.md)、[`資料交換表_單字學習.md`](../API設計/資料交換表_單字學習.md)、[`資料交換表_聽力測驗.md`](../API設計/資料交換表_聽力測驗.md)

| 路由 | 方法 | 模組 | DB 操作 | 實作 |
|---|---|---|---|---|
| `/api/health` | GET | 系統 | `SELECT 1` (ping) | [server.ts](../backend/server.ts) |
| `/api/auth/login` | POST | 認證 | Firebase verify → SELECT/INSERT users → UPDATE last_login_at → 簽 JWT | [routes/auth.ts](../backend/routes/auth.ts) |
| `/api/auth/logout-all` | POST | 認證 | UPDATE users SET token_invalidated_at=NOW() | auth.ts |
| `/api/me` | GET | 使用者 | SELECT FROM users WHERE uid=? | auth.ts |
| `/api/me` | PATCH | 使用者 | UPDATE users SET display_name?/avatar_url? | auth.ts |
| `/api/levels` | GET | 共用 | SELECT level, COUNT(*) FROM words WHERE audio_url IS NOT NULL GROUP BY level | [routes/quiz.ts](../backend/routes/quiz.ts) |
| `/api/quiz/start` | POST | 單字 | SELECT in_progress → 若無：抽 10 + 10×3 干擾 + INSERT session + 10×INSERT questions | quiz.ts |
| `/api/quiz/submit` | POST | 單字 | 對 10 題 UPDATE answer/is_correct → UPDATE session.status='completed' → JOIN words 回 detail | quiz.ts |
| `/api/listening/start` | POST | 聽力 | 依 mode 從 words 或 sentences 抽 → INSERT session + questions | [routes/listening.ts](../backend/routes/listening.ts) |
| `/api/listening/submit` | POST | 聽力 | 對 10 題 UPDATE → UPDATE session → JOIN words/sentences 回 detail | listening.ts |
| ~~`/api/audio/*`~~ | — | — | **已廢除**：App 直連 GCS public URL | — |

### 4.1 認證
- **不查 DB 的 fast path**：每個 request middleware `requireAuth` 只驗 JWT 簽名 + 解 uid（不查 users）
- **撤銷檢查**：JWT 帶 `iat`；若 `user.token_invalidated_at IS NOT NULL AND jwt.iat < that` → 401 TOKEN_EXPIRED
- **JWT 有效期**：30 天

### 4.2 出題演算法（單字測驗）
```
1. 從 words 撈 level=? AND audio_url IS NOT NULL 隨機抽 10 個正答
2. 對每題：
   a. 50/50 決定 direction（zh→truku or truku→zh）
   b. 干擾池 = same level + same category + id ≠ 該詞 + id NOT IN(同 session 其他 9 題正答)
   c. 抽 3 干擾；不足走 fallback（放寬 category → 放寬 dedup）
   d. 4 個 ID shuffle → option_ids（固定順序）
3. INSERT quiz_sessions + 10×quiz_questions
4. Response 帶 prompt_audio_url（GCS URL，**不分 direction 永遠附帶**）
```

### 4.3 出題演算法（聽力測驗）
```
1. 依 mode 決定資料源：
     word_to_zh / word_to_truku → words WHERE level=? AND audio_url IS NOT NULL
     sentence_to_zh             → sentences WHERE level=? AND audio_url IS NOT NULL
   （無 source 過濾：situation/curriculum/essay 都會出）
2. 抽 10 正答
3. 對每題抽 3 干擾：
     word 模式 → 同 level + 同 category
     sentence 模式 → 同 level（不分大類）
4. INSERT listening_sessions + 10×listening_questions
```

### 4.4 邊界處理
| 情境 | 處理 |
|---|---|
| 同 user 同 level 已有 in_progress | 回 200 + 舊 session 題目，`resumed=true`, UPDATE last_active_at |
| 跨等級衝突（送中級但有初級未完成） | 同上 + 額外回 `conflicting_level: "中級"` 讓前端提示 |
| 重複提交 completed session | 比 request 與 DB user_answer：一致回 200 (idempotent)；不一致回 409 SESSION_ALREADY_COMPLETED |
| answers 數 ≠ 10 或 question_id 對不上 | 400 ANSWER_COUNT_MISMATCH / QUESTION_ID_NOT_FOUND |
| 15 天未動 in_progress session | cron 每日掃描 DELETE（CASCADE 自動清 questions）— **目前未排程** |
| Apple 中文姓名 | `display_name = last_name + first_name`（姓在前） |

## 5. 典型資料流（5 個 user-facing flow）

### Flow 1：App 啟動 + Firebase 登入

```
App                              Cloud Run                       Cloud SQL
 │                                  │                                │
 ├─ GET /api/me (Bearer 舊 JWT) ───►│                                │
 │                                  │ verify JWT → uid                │
 │                                  ├─ SELECT * FROM users WHERE uid=?
 │                                  │   AND token_invalidated_at IS NULL ─►│
 │                                  │◄──────────── row 或 null ──────────│
 │◄── 200 {user} 或 401 ────────────│                                │
 │                                  │                                │
 │ (若 401) 顯示登入頁                                                 │
 │                                  │                                │
 │ Firebase SDK → id_token (eyJ…)                                    │
 │                                  │                                │
 ├─ POST /api/auth/login            │                                │
 │  { firebase_token } ────────────►│                                │
 │                                  │ admin.auth().verifyIdToken     │
 │                                  │ → uid, email, name, picture,   │
 │                                  │   sign_in_provider             │
 │                                  ├─ SELECT uid FROM users          │
 │                                  │   WHERE provider=? AND          │
 │                                  │   provider_sub=? ──────────────►│

```

### Flow 2：單字測驗（start → 答題 → submit）

```
App                              Cloud Run                       Cloud SQL

 │                                  │   SELECT … WHERE user_uid=?    │

 │                                  │◄────────── 有/無 ──────────────│
 │                                  │                                │
 │                                  │ ── 若無，跑出題演算法 ──        │
 │                                  ├─ SELECT 10 random words ──────►│
 │                                  ├─ SELECT 干擾 ×10 ──────────────►│
 │                                  ├─ INSERT quiz_sessions ────────►│
 │                                  ├─ INSERT quiz_questions ×10 ───►│
 │                                  │                                │
 │◄── 200 { session_id,             │                                │
 │      questions:[                 │                                │
 │        { question_id, order,     │                                │
 │          direction, prompt,      │                                │
 │          prompt_audio_url ⟶ GCS │                                │
 │          options:[…]×4 } ×10 ] }│                                │
 │                                  │                                │

 │             ×10 } ──────────────►│                                │
 │                                  │ verify uid==session.user_uid   │
 │                                  ├─ SELECT correct_word_id,        │

 │                                  │   WHERE session_id=? ─────────►│
```

## 6. 音檔策略（GCS 直連）

### 6.1 流程

```
┌─ App ─┐                                ┌─ GCS bucket ─┐
│       │                                │ truku-media  │
│ ① 從 API response 取 prompt_audio_url 或 options[].audio_url
│   = https://storage.googleapis.com/truku-media/{path}
│       │                                │              │
│ ② <audio src=url>.play()  或 fetch(url)│              │
├───────────────────────────────────────►│              │
│                                        │ ③ public GET │
│◄── 200 + binary (audio/wav 或 mp3) ────│              │
└─────────────────────────────────────────┘              │
```

### 6.2 為什麼直連 GCS（不走 backend stream）
- 教材來自公開的 [klokah.tw](https://web.klokah.tw)，無敏感資料
- bucket 設 uniform access + allUsers 可讀，CORS=`*`/GET/HEAD
- 免 Cloud Run 頻寬瓶頸與冷啟動延遲
- 後端少一條 route，不必實作 `/api/audio/*`

### 6.3 URL 格式
- DB 內的 `audio_url` 已是完整 https URL
- 規則：`https://storage.googleapis.com/truku-media/{原本機相對路徑}`
- 範例：本機 `字母篇\01_a\音檔\_字母a.mp3` → URL `https://storage.googleapis.com/truku-media/字母篇/01_a/音檔/_字母a.mp3`

### 6.5 `audio_index` 表的角色
- 仍存於 DB 但 **App 流程不查**
- 用途：ETL 對帳、未來若改 stream 模式可重啟用、本機檔案 ↔ DB row 的對照

## 8. 維運：cron、撤銷 token、CORS

### 8.1 Secret Manager（6 個 secret）

| Secret | Cloud Run env | 用途 |
|---|---|---|
| `DB_PASSWORD` | `DB_PASSWORD` | postgres user 密碼 |
| `db-host` | `DB_HOST` | unix socket 路徑 |
| `db-user` | `DB_USER` | DB user（後續應改非 postgres 低權限） |
| `db-name` | `DB_NAME` | `truku` |
| `JWT_SECRET` | `JWT_SECRET` | JWT 簽章 key |
| `cors-origins` | `CORS_ORIGINS` | 逗號分隔網域；`*` 全開（限開發） |

啟動時 `CORS_ORIGINS` 缺值會 throw，避免意外 fallback 成 `*`。

### 8.2 15 天廢棄 session 清理

**OAS 規範**：每日 02:00 執行
```sql
DELETE FROM quiz_sessions
WHERE status='in_progress' AND last_active_at < NOW() - INTERVAL '15 days';

DELETE FROM listening_sessions
WHERE status='in_progress' AND last_active_at < NOW() - INTERVAL '15 days';
```
CASCADE 會自動清 `quiz_questions` / `listening_questions`。

**狀態**：⚠️ **目前未排程**。腳本就緒於 [`backend/scripts/expire_sessions.mjs`](../backend/scripts/expire_sessions.mjs)。

**排程三選一**（上線前必須擇一）：
- **(A) Cloud Scheduler → Cloud Run Job**：包成 Cloud Run Job，Scheduler 每日 02:00 觸發
- **(B) Cloud SQL pg_cron**：開 `cloudsql.enable_pg_cron` flag（需 instance 重啟）
- **(C) HTTP endpoint** `POST /api/admin/expire` + Cloud Scheduler OIDC

### 8.3 JWT 撤銷
- JWT 帶 `iat`（issued at）claim
- Backend 每次驗 token 時：
  ```
  if user.token_invalidated_at IS NOT NULL AND jwt.iat < user.token_invalidated_at:
      回 401 TOKEN_EXPIRED
  ```
- 使用者主動：`POST /api/auth/logout-all` → SET `token_invalidated_at = NOW()`

### 8.4 監控觀察
- Cloud Run logs：`gcloud run services logs read truku-api --region=asia-east1`
- 健康檢查：`curl https://truku-api-jako626zpa-de.a.run.app/api/health` 回 `{ok:true, time}`
